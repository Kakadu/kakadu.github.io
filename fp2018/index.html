Пролог

  • Repl.it! и там выбрать Haskell. На первых порах хватит.

  • Eсть плагин для IntelliJ IDEA

  • Нужно устанавливать Haskell IDE Engine a.k.a. HIE и в README читать как настроить ваш любимый редактор (VsCode, Emacs, etc.). Только убедитесь, чтобы версии совпадали.

➜  haskell-ide-engine git:(master) stack exec ghc -- --version
The Glorious Glasgow Haskell Compilation System, version 8.4.4
➜  haskell-ide-engine git:(master) hie --version
Version 0.4.0.1, Git revision 7e0b19721872d1c8e44f49484ac75f6c61412349 (2220 commits) x86_64 ghc-8.4.4

  • Самая распостраненый способ уставновить Haskell - это Haskell Platform (http://www.haskell.org/platform). Работает для Windows, Linux и Mac. Для курса будет достаточно минимального варианта установки.

  • В Haskell Platform есть простые среды разработки GHCi и WinGHCi. Вот краткая инструкция о том, как начать с ними работать.

Вопросы к экзамену

За мямленье в ответ на вопрос с ☦ будет больно.

  1. Абстрактный спискок и сложность основных операций у него. Какие (минимум две) реализации абстрактных списков Вы знаете и что там со сложностью основных опeраций?

  2. ☦ Алгебра типов. Простейшие типы из которых можно много чего построить.

  3. Эквивалентность, равенство и изоморфизм. И для типов тоже.

  4. Как из простейших типов построить что-то изоморфное данному алгебраическому типу?

  5. ☦ Лямда-исчисление. Три правила перобразования лямбда-термов.

  6. Стратегии вычисления лямбда-термов: call-by-name, call-by-value, call-by-need. Почему одно лучше/хуже другого?

  7. ☦ Унификация и подстановки. ☦ Capture avoiding substitution. Индексы де Брауна.

  8. ☦ Функторы. ☦ Законы функторов.

  9. ☦ Аппликативные функторы. Законы аппликативов.

  10. ☦ Монады. ☦ Законы монад. ☦ Выражение функторов и аппликативов, если монада уже описана. ☦ Стандартные монады: Reader, Writer, ☦ State, ☦ List, ☦ Maybe, Cont, ☦ IO.

  11. ☦ Do-нотация. ☦ Преобразование монадического кода из do-нотации и обратно.

  12. Преобразования кода с list comprehension в do-нотацию и обратно.

  13. Стандартные функции над списками. Будьте готовы написать двухстрочную функцию (map, drop, etc.) не отходя от кассы (или типа того).

  14. Классы типов. Мотивация.

  15. Free monad. Пример.

  16. Фантомные типы. Пример.

  17. Экзистенциальные типы. Почему такое название?

  18. Type families. Мотивация.

  19. ☦ Катаморфизм и анаморфизм. Пример.

  20. Параморфизм. Хиломорфизм.
